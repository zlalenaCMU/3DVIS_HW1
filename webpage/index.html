4. Camera Transformations

Transform 1

This cow is rotated 90 Degrees from the original. So the R_relative matrix is created to reflect this rotation.

Transform 2

This cow is shifted down and to the left. In order to get this transformed cow we must move it left positive 1/2, and down negative 1/2.

Transform 3

This cow is smaller, which means it's further from the camera in the z-direction.

Transform 4

The rear end of this cow is rotated towards us -90 degrees. This means the middle row of our R matrix is 0's and 1's and the first and last row contain our sines and cosines, because we are rotating about the x-axis, instead of the z. Then we have to get the cow back in center of the frame my translating a bit.

Tradeoffs Between Rendering as a Mesh VS a Point Cloud

Point clouds can be rendered faster than meshes depending on the number of points, instead of creating a surface we more just plopping points in 3D which is a lot faster than rendering a mesh. This may longer hold true if we have a large amount of points representing a simple mesh like a cube. This also brings up a quality issue with point clouds, as seen in the plant renderings, because we are just plotting points we have data for the quality isn't great. If we compared it to having a full mesh of a plant where we weren't missing details and the surface was complete our result would look much better. Point clouds can use significantly more memory than meshes because we need to store the position and color (and possible more info) for every point we have. The point cloud Torus was a bit easier to renderer than the mesh in my opinion, but neither was overly complicated.
